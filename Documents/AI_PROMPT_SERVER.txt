## 🎯 PROBLEMS TO FIX

### 🔴 Priority 1: STT High Latency

**Symptom:**
- พูดเสร็จแล้ว → รอ 5-10 วินาที → แปลออกมา
- Gateway ส่ง audio มา → Server ช้าในการ process

**Where to Check:**
- `app/api/stt_ws.py` - WebSocket endpoint
- Faster-Whisper model configuration
- FFmpeg audio pipeline
- Audio buffer size

**Diagnostic Commands:**
```bash
# Check GPU usage during STT
nvidia-smi -l 1

# Monitor WebSocket connections
ss -tuln | grep 8080

# Check model loading time
# (ใน code: faster_whisper.transcribe_audio(...))

# Test with smaller audio chunk
# Current: 3200 bytes (100ms @ 16kHz)
```

**Potential Solutions:**
1. Reduce audio buffer size (ลด latency แต่อาจเพิ่ม CPU usage)
2. Use streaming transcription (real-time instead of batch)
3. Optimize VAD (Voice Activity Detection) settings
4. Add early-stop on silence detection


### 🟡 Priority 2: STT Accuracy

**Issues:**
- "วัวล่า" → "วัวหน้า" (บางครั้ง)
- Background noise ทำให้แปลผิด

**Solutions:**
1. Fine-tune VAD threshold
2. Add noise reduction filter
3. Test different Whisper model size (large-v3 vs medium)


### 🟢 Priority 3: LLM Planner Endpoint

**Issue:**
```
Gateway → POST /plan/plan_from_text → 404 Not Found
```

**Check:**
- `app/api/plan_router.py` - Does endpoint exist?
- Is it registered in `app/main.py`?
- Gateway expects: `/plan/plan_from_text`
