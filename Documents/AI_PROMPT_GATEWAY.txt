### 🟡 Priority 2: Wake Word Detection Delay

**Issue:**
- พูด "วัวล่า ไปข้างหน้า" → รอ 5-10 วินาที → ถึงเห็น log:
  ```
  13:37:54 | 📝 RAW TRANSCRIPT: 'วัวล่า ไป ข้างหน้า'
  13:37:54 | 🎯 COMMAND EXTRACTED: 'ไป ข้างหน้า'
  ```

**Cause:**
- STT บน Server ช้า (ปัญหาหลักอยู่ที่ Server)
- Gateway รอ transcript จาก WebSocket

**Possible Gateway-side Optimization:**
- Add timeout for WebSocket response
- Show "Processing..." feedback earlier
- Buffer management?


### 🟢 Priority 3: LLM Planner Fallback

**Current Behavior:**
```
13:37:55 | 🧠 Trying LLM Planner...
13:37:55 | ⏭️  LLM Planner: No plan generated (404)
13:37:55 | 🎮 Trying Motion Parser...
13:37:55 | ✅ Motion Intent Detected: {...}
```

**Fix:**
- Either implement `/plan/plan_from_text` on Server
- Or remove LLM Planner call (just use Motion Parser)
